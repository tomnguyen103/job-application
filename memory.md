## Problems solved

- **Chat-attached images aren't files.** Sources that exist on disk: `%TEMP%\jobapp-logo2-icon-crop.png` (960×960) + `%TEMP%\jobapp-logo2-text-crop.png` (1420×620) — full-res logo artwork crops, still present. `Get-Clipboard -Format Image` works only if the user pasted (not drag-drop).
- **Background-window screenshots:** inject `8px solid #ff00ff` fixed marker via Chrome MCP `javascript_tool`, then Win32 `PrintWindow(hwnd, hdc, 2)` — works with Chrome BEHIND other windows (Teams steals focus; `AppActivate` is unreliable). Sanitize marker border-walks to 3..20px (overlays can cover a side). Hide `<nextjs-portal>` first. NEVER full-screen `CopyFromScreen` while the user works — it once captured their private Teams chat into the repo (restored from git immediately).
- **SVG text rendering via sharp:** spaces between tspans collapse — use `xml:space="preserve"`. Fonts: "Cascadia Code, Consolas, monospace" and "Segoe UI" resolve on this machine.
- **Stale-image triage:** compare disk SHA1 vs served bytes (`Invoke-WebRequest`) to isolate browser cache from server; `.next/cache/images` may not exist in dev.
- **Chrome MCP:** background tabs freeze (CDP 45s timeouts) — verify server-side instead of fighting; extension redacts JS results containing query-string URLs (return booleans/numbers only).
- Palette quantization (`png({palette:true, quality:100})`) safe for these assets — no banding (verified visually); 153KB→53KB.
